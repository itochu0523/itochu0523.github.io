最終目的

すべての商品（JAN コード）に対して、メーカー・ブランド・カテゴリーを正確に付与する。
各属性は独立したフローで整備し、最終的に商品テーブルへ JOIN してマッピングを完成させる。

dmc-ops.master — 正規化マスター

メーカーコード・メーカー名を正規化（N 対 1）

メーカーコードとメーカー名は N 対 1 の関係。それぞれをユニークキーとした正規化テーブルを作成。

maker_code_master

メーカーコード がユニーク
PK: standard_gs1_company_code

maker_name_master

メーカー名 がユニーク
PK: company_name（名寄せ済み）

Phase 1-B ブランドマスター整備 → 非正規化テーブル構築
5

dmc-ops.master — ブランド付与

maker_name_master にブランド情報を付加

メーカー名に対してブランド名が N 対 1。 maker_name_master を起点にブランドサイト URL を登録し、 DMC 手動作業またはクローリングによりブランド名を生成・紐づける。

ブランドサイト登録

各メーカーのブランドサイト URL を手動で登録

ブランド名生成

DMC 作業 または クローリング でブランド名を取得・付与

↓maker_code を KEY に非正規化テーブルを生成
6

dmc-ops.master — 非正規化テーブル

maker_brand_master を作成

メーカーコードを PK とし、メーカー名・ブランド名・クローリング属性を一本化した非正規化テーブル。 JANコードとのマッチング処理の基盤となる。

dmc-ops.master.maker_brand_master
PK maker_code STRING ≒ standard_gs1_company_code
maker_name STRING メーカー名（名寄せ済み）
brand_name STRING ブランド名
brand_site_url STRING ブランドサイト URL
crawl_attributes JSON クローリング取得属性（拡張用）
created_at / updated_at TIMESTAMP
PK: maker_code maker_name brand_name brand_site_url crawl_attributes (JSON)

※ クローリングで取得できた属性は crawl_attributes JSON カラムに格納。将来的な拡張に対応。

Phase 2 カテゴリーマッピング 検討中
🔧 Work In Progress

カテゴリーフロー — 現在設計中

カテゴリーはメーカー・ブランドとは独立したフローとして設計する。
森井カテゴリマスタ（L1〜L5 階層）を活用し、商品に対して最小分類（L5）を付与することが目標。 具体的なマッピングロジックは現在検討中。

A カテゴリーマスター整備（森井カテゴリマスタ L1〜L5 階層の確定）
B 商品 → カテゴリーのマッピングロジック設計（ルールベース / ML / 手動）
C マッピング結果の格納先テーブル設計
D メーカー・ブランドとカテゴリーの統合マッピングテーブル設計
